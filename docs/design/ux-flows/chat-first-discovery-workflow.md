# Chat-first Discovery workflow

Дата: 2026-07-08

## Цель

Новый Discovery Workspace должен сделать AI Discovery Chat основной точкой входа в работу над проектом. Пользователь формулирует намерение в чате, AI предлагает изменения через `proposed_patch`, интерфейс показывает предпросмотр, а structured forms остаются источником истины для проверки, ручного редактирования и подтверждения результата.

## Базовый сценарий

1. Пользователь открывает Project Workspace.
2. Левая навигация показывает этапы, readiness, устаревшие артефакты и блокировки.
3. Пользователь пишет задачу в правой AI Chat Panel.
4. AI отвечает текстом и, если требуется изменение артефакта, создаёт proposed patch.
5. Правая панель показывает preview/diff, целевой артефакт, причины и предупреждения.
6. Центральная область показывает structured state текущего артефакта.
7. Пользователь открывает целевой этап, сравнивает предложение с формой и вручную редактирует при необходимости.
8. Пользователь нажимает «Применить в артефакт».
9. Артефакт обновляется, readiness/completion пересчитываются, зависимые этапы получают статус `stale` при необходимости.

## Правила взаимодействия

- Chat инициирует работу, но не применяет изменения автоматически.
- Любое изменение structured artifact проходит цепочку: `proposed_patch` → preview/diff → apply.
- Формы этапов всегда видимы и доступны для ручной правки.
- AI Chat должен уметь объяснить, какой артефакт будет изменён и почему.
- Если данных недостаточно, AI предлагает уточняющие вопросы вместо генерации уверенного результата.
- Если source evidence слабый, UI показывает предупреждение рядом с предложением AI и в центральной форме.

## Основные пользовательские действия в AI Chat

| Действие | UX-поведение | Результат |
|---|---|---|
| Сформулировать артефакт | AI готовит proposed patch для текущего или выбранного этапа. | Preview/diff в правой панели. |
| Применить в артефакт | Доступно только при pending proposed patch. | Apply patch, обновление формы и версии. |
| Открыть этап | Переводит центральную область на целевой артефакт. | Пользователь проверяет structured state. |
| Показать источники | Открывает Context/source trace или drawer источников. | Пользователь видит evidence. |
| Задать уточняющие вопросы | AI формирует список вопросов по текущему gap. | Вопросы попадают в stage questions. |
| Проверить качество | Запускает validation для текущего этапа или Final BT. | Warnings/errors в форме и чате. |
| Экспортировать DOCX | Доступно для Final BT или готового проекта. | Экспорт документа. |

## Состояния

| Состояние | Где видно | Что делает пользователь |
|---|---|---|
| `empty` | Левая навигация, центральный empty state | Даёт задачу AI или заполняет форму. |
| `draft` | Badge этапа, версия артефакта | Уточняет и сохраняет данные. |
| `proposed_patch` | Правая AI Chat Panel | Смотрит preview/diff. |
| `applied` | Правая панель и версия артефакта | Проверяет форму и продолжает workflow. |
| `stale` | Левая навигация, предупреждение в центре | Обновляет зависимый этап через AI. |
| `blocked` | Readiness/status panel | Добавляет источники или отвечает на вопросы. |
| `ready` | Readiness, completion, stage badge | Переходит к следующему этапу. |

## Handoff для frontend

- Основной layout: `left 240-280px`, `center minmax(0, 1fr)`, `right 340-420px`.
- Правая панель sticky на desktop; на tablet/mobile переходит под центральную область.
- Левая навигация должна показывать stage status, readiness и stale markers без скрытия этапов.
- Центральная область не должна быть заменена чат-интерфейсом: это structured state artifact.
- Все user-facing тексты на русском языке.
