# Пользовательская инструкция AI Discovery Platform

Дата: 2026-05-17

## Для кого инструкция

Инструкция предназначена для Product Owner, бизнес-аналитика и системного аналитика, которые проводят Discovery и готовят БТ с помощью AI Discovery Platform.

## Основной сценарий

1. Откройте платформу.
2. Создайте проект Discovery.
3. Заполните ручной контекст: название инициативы, цель, домен, владелец процесса, связанные системы.
4. Добавьте источники: документы, таблицы, PDF, ссылки.
5. Нажмите «Обновить контекст».
6. Проверьте readiness, coverage и missing information.
7. Ответьте на уточняющие вопросы AI.
8. Сгенерируйте этап «Проблема».
9. Проверьте источники и скорректируйте формулировку.
10. Сгенерируйте «Цель» и «Бизнес-эффект».
11. Подготовьте AS IS, TO BE, Use Cases и требования.
12. Запустите проверку полноты.
13. Соберите финальный БТ.
14. Экспортируйте DOCX.

## Работа с контекстом

Поддерживаемые форматы:
- TXT
- MD
- CSV
- DOCX
- PDF
- XLSX

Если источник не обработан:
- проверьте формат;
- проверьте размер файла;
- замените XLS на XLSX или CSV;
- загрузите файл повторно;
- добавьте ручное описание, если текст не извлекается.

## Readiness

Readiness показывает, достаточно ли контекста для AI-генерации. Низкая готовность не всегда блокирует работу, но результат потребует большего ручного уточнения.

## Source trace

Source trace показывает, на какие источники и chunks опирался AI. Если важное утверждение не имеет источника, его нужно проверить вручную.

## LLM settings

Режимы:
- Mock: безопасный режим без внешнего LLM.
- OpenRouter: внешний provider, требует API key.
- Корпоративный OpenAI-compatible provider: предпочтительный режим для закрытого контура.

API key не должен передаваться в документах, Trello или сообщениях.

## Экспорт DOCX

Перед экспортом проверьте:
- все обязательные этапы заполнены;
- validation report не содержит критичных пробелов;
- источники указаны;
- открытые вопросы либо закрыты, либо явно вынесены в раздел рисков.

## Ограничения

- AI-результат является черновиком и требует экспертной проверки.
- Документы с prompt injection считаются недоверенными.
- Внешние LLM нельзя использовать для закрытых данных без разрешения.
