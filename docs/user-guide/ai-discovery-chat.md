# AI Discovery Chat

AI Discovery Chat поддерживает два разных сценария: ответить по источникам и подготовить patch для артефакта.

## Вопросы по источникам

Задавайте обычные вопросы по загруженным документам, ссылкам и context sources:

- `найди описание БТ во вложении`;
- `что в документе сказано про автопролонгацию ИБС?`;
- `найди требования в загруженном файле`;
- `есть ли в БТ описание процесса?`;
- `покажи, где в источниках написано про ИБС`.

В этом режиме AI Chat ищет evidence в `uploaded_files`, `documents`, `links`, `chunks` и `extracted_text`. Если подтверждение найдено, ответ содержит краткую выдержку и блок «Найдено в источниках». Если подтверждение не найдено, Chat честно сообщает, что в загруженных источниках его нет.

## Patch-flow

Команды и запросы на изменение артефактов, например `@problem`, `@goal`, `@requirements`, создают proposed patch. Такой patch не применяется автоматически: сначала появляется preview, затем пользователь нажимает «Применить в артефакт».

Обычный вопрос по источникам не создаёт patch и не показывает preview изменения.

## Контекст требует обновления

После загрузки файла контекст может иметь статус `requires_update`. Это значит, что полная выжимка контекста ещё не пересобрана. Если chunks или extracted text уже есть, AI Chat использует их для ответа и показывает предупреждение, что контекст требует обновления.

## Проверка в браузере

1. Откройте проект.
2. Загрузите DOCX на этапе «Контекст».
3. Спросите в AI Chat: `найди описание БТ во вложении`.
4. Проверьте, что ответ содержит выдержку, evidence/source trace видны, а action для применения patch отсутствует.
