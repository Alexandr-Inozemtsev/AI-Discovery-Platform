# Глоссарий слоев AI-агентов

| Слой | Назначение | Пользователь | Где находится | Примеры агентов | Что нельзя путать |
|---|---|---|---|---|---|
| Product AI Agents | Продуктовые агенты, которые помогают будущим пользователям AI Discovery Platform выполнять Discovery: анализировать контекст, формулировать проблему, цель, требования и проверять качество артефактов. | Пользователь продукта. | `discovery-ai-agent/backend/app/agents`, `discovery-ai-agent/backend/app/api`, `discovery-ai-agent/frontend`. | `ContextIngestionAgent`, `ProblemAgent`, `GoalAgent`, `BusinessEffectAgent`, `RequirementsAgent`, `CriticAgent`. | Нельзя считать их Codex-ролями разработки или подключать глобальных `ai-*` delivery agents как backend-классы продукта. |
| Global Codex Delivery Agents | Глобальные роли разработки, которые помогают через Codex проектировать, реализовывать, проверять и выпускать AI Discovery Platform. | Владелец/разработчик проекта в Codex. | Глобально: `C:\Users\alexp\.codex\agents`, `C:\Users\alexp\.agents`; в репозитории только операционные документы `docs/ai-delivery-agents`. | `ai-orchestrator`, `ai-backend-developer`, `ai-frontend-developer`, `ai-qa-engineer`, `ai-trello-analyst`. | Нельзя считать их runtime-функциональностью продукта или сервисами FastAPI. |
| LLM Provider | Внешний или корпоративный поставщик LLM (large language model, большая языковая модель), который выполняет генерацию текста по prompt. | Product AI Agents, backend runtime, иногда Codex при разработке. | Конфигурация backend/окружения, provider adapter, secrets-хранилище вне репозитория. | OpenAI-compatible provider, corporate gateway, local model endpoint. | Нельзя путать provider с агентом: provider не владеет бизнес-логикой, backlog или handoff. |
| Agent Runtime внутри backend | Технический механизм запуска продуктовых агентов приложения: контракты запуска, результат, ошибки, trace metadata, readiness и fallback. | Backend приложения и продуктовые агенты. | `discovery-ai-agent/backend/app/agents`, `discovery-ai-agent/backend/app/services`, `docs/architecture/agent-runtime-roadmap.md`. | `AgentOrchestrator`, `BaseAgent`, `AgentContext`, `AgentResult`. | Нельзя описывать здесь глобальные Codex delivery agents или Trello/Gantt процесс разработки. |
| Codex Orchestrator как управляющий промпт разработки | Координирующий prompt/роль, которая разбирает задачу пользователя в Codex, выбирает delivery agents, задает порядок работ, gates и итоговый отчет. | Владелец проекта в Codex. | `docs/ai-delivery-agents/03-codex-orchestrator-prompt.md`, `docs/ai-delivery-agents/agents/ai-orchestrator.md`. | `ai-orchestrator` как управляющий профиль разработки. | Нельзя считать его backend `AgentOrchestrator` продукта или продуктовым API endpoint. |
