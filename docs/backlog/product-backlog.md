# Продуктовый backlog AI Discovery Platform

Дата: 2026-05-17

## Назначение backlog

Backlog фиксирует целевую декомпозицию развития AI Discovery Platform до промышленного корпоративного контура. Документ является источником для Trello-доски, архитектурных решений, БТ, ТЗ и последующей delivery-декомпозиции. Код на этом этапе не меняется.

## Правила декомпозиции

- Иерархия: ЭПИК -> ФИЧА -> USER STORY -> ЗАДАЧА -> ПОДЗАДАЧА.
- Все названия и описания написаны на русском языке.
- Технические имена существующих сущностей, например `AgentRuntime`, `SimpleRetriever`, `source_trace`, `ArtifactType`, не переводятся.
- Приоритеты: `P0` критично для MVP, `P1` нужно для MMP, `P2` важно для целевого контура, `P3` улучшение.
- Этапы: MVP, MMP, Целевой контур.

## MVP

В MVP входят сквозной Discovery-процесс, AI Discovery Chat как единая точка входа, управление проектами, контекст, извлечение текста, SimpleRetriever без внешних зависимостей, единый контракт `AgentResult`, `StageProcessorRequest/StageProcessorResult`, генерация ключевых этапов, финальный БТ, DOCX export, базовая трассировка источников, LLM settings, `ToolPolicy`, базовая безопасность, инструкция запуска и Trello operating model.

## MMP

В MMP входят расширенные этапы AS IS/TO BE/Use Cases/NFR, версионирование источников и артефактов, audit log, LLM run log, стабильный UI Kit, метрики качества AI, CI/CD pipeline, расширенные тесты, ролевые сценарии и документация для пилотной эксплуатации.

## Целевой контур

В целевой контур входят adapter boundary под LlamaIndex/Haystack, пилот LangGraph после стабилизации runtime, продвинутый governance LLM, коммерческая упаковка, мониторинг качества AI, отказоустойчивость, расширяемые интеграции и готовность к монетизации.

### ЭПИК-01 Продуктовый фундамент и сквозной Discovery-процесс

Описание: оформить платформу как единое chat-first рабочее пространство Discovery от создания проекта до готового БТ.
Бизнес-ценность: снижает ручной труд Product Owner и аналитика, делает результат Discovery повторяемым.
Зависимости: БТ, ТЗ, UI/UX, модель данных.
Критерии приёмки: пользователь управляет workflow через AI Discovery Chat, видит проект, текущий этап, готовность, источники, артефакты, preview изменений и следующие действия.
Definition of Done: flow описан, покрыт acceptance criteria и отражён в Trello.
Trello: `01 Продуктовый backlog`; labels: Продукт, MVP.
